# ClipTune

ClipTune 是一个基于 Kotlin + Jetpack Compose 的 Android 音频处理应用。  
当前版本采用“目录授权驱动”的媒体库模式，支持音轨提取、前后台统一播放、前台视频展示与后台音频续播。

## 核心能力

- 通过 SAF 授权并读取内部共享存储目录（原始目录 + 输出目录）
- 维护“原始文件列表”和“提取文件列表”
- 从视频提取 MP3（有损）或原始音轨（无损拷贝）
- 内置播放器（`MediaSessionService + MediaController` 前后台统一）：
  - 播放 / 暂停 / 停止
  - 上一首 / 下一首（仅当前列表内）
  - 播放进度拖动
  - 音量调节
  - 循环模式：不循环 / 单曲循环 / 列表循环（默认）
  - 定时关闭：15/30/60 分钟 + 自定义分钟，到点仅暂停播放
- 视频播放（原始列表）：
  - 前台全屏播放器页可展示视频画面
  - 切到后台后继续音频播放，可在通知栏控制
- 文件物理删除（失败会提示原因）

## 本次 UI 优化

- 主页播放器由“大卡片”改为底部“迷你播放器”，显著减少占位。
- 点击迷你播放器可进入全屏播放器页：
  - 视频区域（仅原始列表视频）
  - 完整播放控制
  - 循环模式切换
  - 定时关闭设置与取消
- 列表点击“播放”后不自动跳转全屏，保持当前列表浏览体验。

## 后台播放说明

- 后台播放由 `MediaSessionService`（`PlaybackService`）提供。
- 前后台使用同一播放器内核（Media3 ExoPlayer）。
- 切到后台后可通过通知栏继续控制播放，回到前台状态会同步。

## 技术栈

- Kotlin + Coroutines
- Jetpack Compose + Material3
- FFmpegKit（音轨提取）
- Room（目录配置与媒体索引持久化）
- Media3 ExoPlayer + MediaSessionService + PlayerView（前后台统一播放 + 前台视频渲染）

## 目录与权限

### 目录模式

- 原始目录：扫描视频源文件
- 输出目录：存放提取后的 MP3 / 原始音轨文件
- 目录权限通过 `ACTION_OPEN_DOCUMENT_TREE` 授权并持久化
- 目录更新后会自动刷新媒体列表（无需重启应用）

### Android 权限

- `READ_EXTERNAL_STORAGE`（Android 12 及以下）
- `READ_MEDIA_VIDEO`（Android 13+）
- `READ_MEDIA_AUDIO`（Android 13+）
- `FOREGROUND_SERVICE`
- `FOREGROUND_SERVICE_MEDIA_PLAYBACK`

## 使用流程

1. 进入应用，默认看到目录设置卡片与媒体列表。
2. 选择原始目录、输出目录（支持后续再次切换）。
3. 目录绑定后自动刷新文件库；也可手动点击“刷新文件库”。
4. 在“原始文件”中可播放、提取、删除；在“提取文件”中可播放、删除。
5. 底部迷你播放器可快速控制播放；点击迷你播放器进入全屏播放器页。
6. 全屏页可查看视频画面（原始视频）、切换循环模式、设置定时关闭。
7. 切后台后继续播放，可在通知栏控制。

## 常见问题

### 1) 播放器会占用太多首页空间吗？

不会。首页仅保留迷你播放器；完整控制在全屏播放器页中。

### 2) 后台播放还在吗？

在。后台播放由 `PlaybackService` 提供，通知栏可继续控制。

### 3) 目录选了但列表为空

- 确认目录选择正确
- 点击“刷新文件库”
- 确认目录内有对应类型文件（视频/音频）

### 4) 删除失败

- 可能是目录授权失效或文件已被移动/删除
- 重新选择目录并刷新后重试
